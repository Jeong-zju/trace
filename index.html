TRACE: Trajectory-Routed Causal Memory for Delayed-Evidence Visuomotor Imitation

TRACE stores task evidence while it is visible, routes that evidence through a path-signature memory, and reads it back when the robot reaches an ambiguous branch.

Policy families Regression + Diffusion

Abstract

Long-horizon robot manipulation can require a robot to choose between task alternatives using evidence that is no longer visible. TRACE adds a compact online memory to visuomotor imitation policies. It writes visual-state evidence into fixed-size latent slots, organizes access with streamed path and delta signatures, and exposes the memory through lightweight adapters while leaving the policy backbone, action head, and imitation objective unchanged.

Problem

Current observations can match, while correct actions diverge.

01 Early cue

The robot sees evidence such as object origin, garment side, or tray source.

02 Shared transit

The cue leaves view while different histories pass through similar observations.

03 Ambiguous branch

The next action depends on the causal history rather than the current frame alone.

04 Memory readout

TRACE conditions the action policy on the stored evidence at decision time.

xt-w+1:t ≈ x't-w+1:t
a*t(Ht) ≠ a*t(H't)

A short observation window contains the latest information, but not necessarily the decisive early evidence. The paper frames this as a delayed-evidence imitation setting where the policy needs a fixed-size causal history summary.

Method

A routed memory layer around unchanged imitation policies.

Write

Visual-state evidence is stored when the cue is visible.

Current RGB and proprioceptive features form the memory content, so the module stores the evidence before it disappears from the camera view.

Task Suite

Five tasks ask the same question: can the policy remember the missing cue?

Tool

Object identity determines a later tool sequence.

Initial identity is visible early, then must guide a later sequence choice.

Book

Origin determines route and placement.

The book travels through a similar path, but the final place depends on origin.

Laundry

Origin side chooses brush or fold.

Garments reach a common workspace before the correct branch is selected.

Cable

Source side determines the matched device.

Similar cable observations hide which functional source should be remembered.

Medicine

Tray origin controls box selection and return.

The same object must be associated with its earlier tray through later decisions.

Implementation

The codebase mirrors the paper split: updater, adapter, and deployment path.

StreamingACT

Regression adapter

lerobot_policy_streaming_act exposes path signatures, delta signatures, visual prefix memory, and signature-indexed slot memory for action chunking.

PrismDiffusion

Diffusion adapter

lerobot_policy_prism_diffusion serializes the same memory controls and maps the memory readout into diffusion conditioning.

Runtime

Online signatures

Deployment preprocessing accepts observation.path_signature and observation.delta_signature, matching the paper's causal online interface.
